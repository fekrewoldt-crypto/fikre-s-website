## Overview
The repository now includes a **clinical‑grade personal health record** backend built on Express and SQLite, with secure authentication, encrypted data storage, audit logging, a protected timeline API, and heatmap state synchronization.

## Security Highlights
- **JWT authentication** with short‑lived access tokens and secure refresh cookies.
- **AES‑256‑GCM encryption** of health records; key stored only in `DATA_ENC_KEY` env var.
- **Helmet** for HTTP header hardening.
- **Express rate limiting** on `/api/*` and auth endpoints (default 20 req/min).
- **Audit logging** of read/export actions for compliance.
- **Demo mode** remains isolated; no real data is written when `DEMO_MODE=true`.

## How to Run
```bash
npm install               # install new deps (better-sqlite3, bcrypt, jsonwebtoken, helmet, express-rate-limit)
mkdir -p data               # ensure the SQLite directory exists
npm start                  # launches the server on http://localhost:3000
```
The server will expose:
- `POST /auth/register`
- `POST /auth/login`
- `GET /timeline` (protected)
- `GET /timeline/export` (protected)
- `POST /api/heatmap` (protected)
